## Decisions made
- `parseHhmm(String)` added to `date_utils.dart` as a package-internal helper (not in the spec but needed by both `TimeRange.startMinutes/endMinutes` and `availability_service.dart` for exception/booking overlap checks). This is the natural "shared helpers" location per spec.
- `WeeklyAvailability.empty(uid)` implemented as a `const` named constructor (not a factory) because Dart const constructors may take parameters; this enables the empty instance to be a compile-time constant when called with a literal uid, satisfying the spec's `const` annotation.
- `StudioSettings.initial()` similarly a `const` named constructor with `const <int>{}` / `const <String>{}` field initializers.
- `Slot` equality uses only (trainerUid, ymd(date), start), not end — per spec. This is intentional for deduplication when overlapping windows produce the same slot start.
- `availableSlotsProvider` uses `Provider.family` (synchronous, returning `AsyncValue<List<Slot>>`) rather than a `StreamProvider.family` because it must combine four separate stream providers. The pattern follows the spec exactly.
- Record type `({String trainerUid, DateTime day})` used as `Provider.family` and `StreamProvider.family` key — valid in Dart 3 (structural equality by default).
- `firestore.rules` extended with 4 new collection rules appended before the existing catch-all `/{document=**}` deny rule.
- 27 new l10n keys added to both `app_en.arb` (English) and `app_sr.arb` (Serbian); `flutter gen-l10n` regenerated all three generated files.
